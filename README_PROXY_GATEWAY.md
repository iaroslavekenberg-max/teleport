# Proxy Gateway: Browser without local apps

Этот вариант подходит, если сотрудникам нельзя ставить VPN-клиенты на ПК.

Идея:
1. У тебя на сервере работает шлюз `HTTP proxy -> VLESS`.
2. Сотрудник в браузере использует обычный proxy (host:port + логин/пароль).
3. Внутри сервера трафик идет в твой VLESS.

## Важное ограничение

Этот MVP использует `HTTP proxy` на порту `3128`.
Логин/пароль не шифруются сами по себе. Для продакшена держи сервер в защищенной сети/по allowlist IP.
Если нужно, следующим шагом сделаем TLS-обертку (HTTPS proxy).

## Файлы

- `proxy-gateway/docker-compose.yml`
- `proxy-gateway/.env.example`
- `proxy-gateway/scripts/build-config.mjs`

## Быстрый запуск

1. Перейти в папку:

```bash
cd proxy-gateway
```

2. Создать `.env` из примера и заполнить:

```bash
cp .env.example .env
```

Поля:
- `VLESS_URL` - твой рабочий `vless://...`
- `PROXY_USER` - логин прокси
- `PROXY_PASS` - пароль прокси
- `PROXY_PORT` - обычно `3128`

3. Сгенерировать `config.json`:

```bash
node ./scripts/build-config.mjs
```

4. Запустить шлюз:

```bash
docker compose up -d
```

5. Проверка:

```bash
docker compose logs -f
```

## Что дать сотруднику

- Адрес прокси: `SERVER_IP`
- Порт: `3128`
- Логин: `PROXY_USER`
- Пароль: `PROXY_PASS`

## Как подключиться в Chrome

Есть 2 пути:

1. Через корпоративную политику прокси (предпочтительно для компании).
2. Через расширение-переключатель прокси (если разрешено политиками).

В обоих случаях данные одинаковые: host/port/login/password.

## Как связать с Telegram-ботом

Сейчас у тебя бот выдает `vless://...`.
Для корпоративных пользователей лучше выдавать отдельную команду, например `/corp_proxy`, где бот отправляет:

- `proxy_host`
- `proxy_port`
- `proxy_user`
- `proxy_pass`

Так пользователю не нужен VLESS-клиент вообще.
